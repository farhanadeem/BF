Initialization Fault (INI) Class

Definition

    We define Initialization Fault (INI) as:
    The software uses an incorrect value because a resource was not properly initialized.

Type

    Low (language-related).

Taxonomy

    Fig. 1 depicts INI causes, attributes and consequences.

Attributes

    The attributes of INI are:
    Resource – Variable, Field, Member, Connection, Array, Pointer, etc.
    This is the thing being initialized. It could be a variable, a member of a structure or record, parts of an array, a pointer or reference, disk file, entropy pool, network connection, etc.
    Value – Small, Medium, Large.
    This indicates the incorrect value. It could be several bytes or even as small as one byte of memory; or as large as an entire database.
    Problem – Uninitialized, Improperly Initialized.
    This indicates the problem leading to use of incorrect value. It covers both not initialized at all and not initialized correctly. The resource may not be initialized at all for many reasons: the programmer neglected to write initialization at all, program execution followed an unexpected path, the initialization routine was not called, etc. Not initialized correctly includes initialized with a value that leads to a security concern, see CWE-457.
    Strings in C must have a NULL at the end. Creating a C string without a terminating NULL is a particular danger.
