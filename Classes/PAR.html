Pointer Arithmetic Fault (PAR) Class

Definition

    We define Pointer Arithmetic Fault (PAR) as:
    Software produces a faulty value because of incorrect pointer arithmetic.

Type

    Low (language-related)

    Fig. 1 depicts PAR causes, attributes and consequences.

Attributes

    The attributes of PAR are:
    Fault –
  • Numeric value with operator (that is, operator used incorrectly) (have different faults for relational and arithmetic operators?)
  • Incorrect Operator (no correct way to use this operator, e.g. * or /)
  • Operation between pointers that are not elements of the same array object (see C11, Section 6.5.6 Additive operators, para. 9ff)

Notes

    This is class was first defined for SATE VI Ockham.
    Using relational or arithmetic operators on a pointer and a numeric type is included. Here is context code for following examples of PAR:
    char data[10], otherDatap[5], *datap;
    datap = fgets(data, 10, infile);
    Relational operator and numeric value:
    if (datap < 0) {
    Comparing pointers that are not to elements of the same array object:
    if (datap+2 > otherDatap) {
    Subtracting pointers that are not to elements of the same array object:
    int diff = datap - otherDatap;
    Multiplication:
    int value = datap * 2;
    Adding pointers:
    char *sump = datap + otherDatap;
    Adding non-integer type:
    char *nextp = datap + 3.1415926535;
    Consider the following:
    datap = fgets(data, 10, infile);
    if (datap < 0) {
    The conditional statement has a chain where Pointer Arithmetic Fault (PAR) causes Unchecked Error (UCE).
