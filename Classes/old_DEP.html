Dereference Erroneous Pointer (DEP) Class

Definition

    We define Dereference Erroneous Pointer (DEP) as:
    Software produces a faulty value because of pointer misuse.
    A pointer should refer to a particular place in memory consistent with the type of the pointer. If the value of the pointer ("address") is wrong, then dereferencing the pointer may return a garbage value. (The execution engine may catch such an invalid dereference.) Possible reasons (causes?) that the value is wrong are:
    NULL
    • Pointer scaling arithmetic was done wrong
    • Arithmetic based on assumed data or memory layout
    • Other incorrect math, such as ARC or WOP
    • Dereference of an address supplied by an untrustworthy source
    Note: Passing an erroneous pointer to free() or realloc() is MAL, not DEP.
    Note: Use after free is MAL, not DEP. One could argue that the pointer was once valid (not erroneous), but no longer is.
    Note: Use of uninitialized pointer is INI, not DEP.

Type

    Low (language-dependent)

Attributes

    The attributes of DEP are:
    Fault –
    • NPD (NULL Pointer Dereference)
    • Incorrectly computed pointer is dereferenced
    • Untrusted Pointer Dereference
    Access – Read, Write.
