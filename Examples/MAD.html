Memory Addressing Bugs (MAD) Examples

CVE-2017-17833

BF Taxonomy

    MAD leads to MAL
    Common Attributes:
      Source Code: Codebase (slpd_process.c)
      Execution Space: Userland
      object Location: Heap
    MAD
    Cause: Misssing Operation
    Attributes:
      Operation: Reposition
      Owner: Pointer (result)
      object: Data Structure (array *result)
    Consequence: Dangling Pointer
    MAL
    Cause: Dangling Pointer
    Attributes:
      Operation: Reallocate (realloc(*send_buf))
      Owner: Pointer (send_buf & result)
      object: Data Structure (array *result)
      Mechanism: Explicit
      Owners: Multiple
    Consequence: Double Free, which could lead to arbitrary code execution and DoS.

BF Description:

    CVE Description

    Analysis

      OpenSLP releases in 1.0.2 and 1.1.0 code streams have a bug is in the slpd_process.c file. The result pointer is set to own to the same object as sendbuf in the ProcessSrvRqst() function. To increase the object’s size, sendbuf is used in a reallocation routine, which is called in the ProcessDASrvRqst() function. This reallocation leaves the result pointer dangling. Later, the result pointer in turn is used for reallocation and causes double free.
